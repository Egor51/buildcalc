## 🔴 Критические проблемы (высокий приоритет)

### 1. Time to Interactive (TTI) - 5.7 сек (score 0.68)
**Проблема**: Страница становится интерактивной слишком поздно.

**Решения**:
- ✅ Оптимизировать загрузку JavaScript (code splitting)
- ✅ Отложить загрузку некритичного JS (lazy loading)
- ✅ Минимизировать размер бандлов
- ✅ Использовать динамические импорты для тяжелых компонентов
- ✅ Оптимизировать Google Tag Manager (загружать асинхронно)

**Ожидаемый эффект**: Улучшение TTI до ~3-4 сек, score до 0.85+

---

### 2. Неиспользуемый JavaScript - 347 КБ (score 0.5)
**Проблема**: Загружается много неиспользуемого кода.

**Основные источники**:
- Google Tag Manager: ~60 КБ неиспользуемого
- GTM gtag.js: ~60 КБ неиспользуемого
- Next.js chunks: ~24 КБ неиспользуемого (`3ea06bf7f88bf46e.js`)

**Решения**:
- ✅ Включить tree-shaking в production build
- ✅ Использовать динамические импорты для компонентов, которые не нужны сразу
- ✅ Оптимизировать загрузку GTM (использовать `strategy="lazyOnload"` вместо `afterInteractive`)
- ✅ Проверить и удалить неиспользуемые зависимости
- ✅ Использовать `next/dynamic` для компонентов ниже fold

**Ожидаемый эффект**: Сокращение размера JS на 100-150 КБ, улучшение FCP и TTI

---

### 3. Render-blocking ресурсы (score 0.5)
**Проблема**: CSS блокирует рендеринг страницы.

**Ресурс**: `/_next/static/chunks/ec5c93706999618a.css` (10 КБ)

**Решения**:
- ✅ Критический CSS инлайнить в `<head>`
- ✅ Остальной CSS загружать асинхронно
- ✅ Использовать `next/font` для оптимизации шрифтов (уже используется)
- ✅ Минифицировать CSS в production
- ✅ Разделить CSS на критические и некритические части

**Ожидаемый эффект**: Улучшение FCP на 200-400 мс, score до 0.9+

---

### 4. Inspector Issues - CSP проблемы (score 0)
**Проблема**: Обнаружены проблемы с Content Security Policy.

**Решения**:
- ✅ Настроить правильный CSP заголовок
- ✅ Разрешить необходимые источники для GTM, шрифтов, стилей
- ✅ Использовать nonce или hash для inline скриптов

**Ожидаемый эффект**: Устранение предупреждений в консоли, улучшение безопасности

---

## 🟡 Важные оптимизации (средний приоритет)

### 5. First Contentful Paint (FCP) - 2.4 сек (score 0.7)
**Проблема**: Первый контент появляется поздно.

**Решения**:
- ✅ Оптимизировать серверный рендеринг (SSR)
- ✅ Использовать статическую генерацию (SSG) где возможно
- ✅ Предзагружать критические ресурсы (`<link rel="preload">`)
- ✅ Оптимизировать изображения (использовать `next/image`)
- ✅ Минимизировать размер HTML ответа
- ✅ Использовать HTTP/2 Server Push для критических ресурсов

**Ожидаемый эффект**: Улучшение FCP до ~1.5-1.8 сек, score до 0.9+

---

### 6. Speed Index - 4.1 сек (score 0.79)
**Проблема**: Контент появляется медленно.

**Решения**:
- ✅ Оптимизировать загрузку шрифтов (использовать `font-display: swap`)
- ✅ Использовать skeleton screens для компонентов
- ✅ Предзагружать данные для следующей страницы
- ✅ Оптимизировать анимации (использовать `will-change`, `transform`)
- ✅ Уменьшить количество ре-рендеров

**Ожидаемый эффект**: Улучшение Speed Index до ~2.5-3 сек, score до 0.9+

---

### 7. Accessibility Score - 0.84
**Проблема**: Есть проблемы с доступностью.

**Решения**:
- ✅ Проверить все интерактивные элементы на наличие `aria-label`
- ✅ Убедиться, что все формы имеют правильные `label`
- ✅ Проверить контрастность цветов (WCAG AA минимум)
- ✅ Добавить `skip to content` ссылку
- ✅ Улучшить навигацию с клавиатуры
- ✅ Добавить `role` атрибуты где необходимо

**Ожидаемый эффект**: Улучшение Accessibility score до 0.95+


### 4. CSP Headers
```typescript
// В next.config.mjs
headers: async () => [
  {
    source: '/:path*',
    headers: [
      {
        key: 'Content-Security-Policy',
        value: "default-src 'self'; script-src 'self' 'unsafe-inline' https://www.googletagmanager.com; ..."
      }
    ]
  }
]
```
